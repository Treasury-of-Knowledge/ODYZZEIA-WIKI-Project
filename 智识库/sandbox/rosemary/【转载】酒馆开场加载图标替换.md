## 自定义
方法

```
:root {
    --custom-loader-img: url("这里按格式填")}
```

按照要求填入你想要的格式,支持图床和base64,本地文件我没试过

注意最好用扣了背景的

```
1. 图床或网络图片（绝对链接）
如果你把图片上传到了图床，直接把带 http:// 或 https:// 的完整直链丢进去就行。

CSS
:root {
  /* 注意：必须是图片的直链，通常以 .png, .jpg, .webp 或 .gif 结尾 */
  --custom-loader-img: url("https://example-image-host.com/my-loader.gif");
}
2. 本地图片（项目内路径）
如果你是在写前端项目（比如 Vue 或 React），通常会根据 CSS 文件和图片文件的相对位置来写路径。

相对路径（相对于当前这个 CSS 文件的位置）：

CSS
:root {
  /* ./ 代表当前目录，../ 代表上一级目录 */
  --custom-loader-img: url("../assets/images/loader.gif"); 
}
绝对路径（相对于网站的根目录或 Vite/Webpack 配置的 public 目录）：

CSS
:root {
  /* 以 / 开头，代表从网站根目录开始找 */
  --custom-loader-img: url("/images/loader.gif"); 
}
```
